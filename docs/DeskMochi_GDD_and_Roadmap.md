## 一、项目概述 (Project Overview)

DeskMochi 是一款兼具陪伴感与生产力属性的桌面悬浮宠物应用。产品以“软糯可交互”的物理反馈为核心情绪价值，结合轻量级的时间管理工具与开发者专属的数据统计功能。整体调性主打简单直白、软萌亲切，旨在提供无压力的陪伴体验，无缝融入用户的日常工作流中。

## 二、核心系统与玩法设计

### 1. 软体交互与自定义系统

| 功能模块 | 具体表现 | 实现方案简述 |
| --- | --- | --- |
| 物理交互反馈 | 鼠标拖拽、揉捏、戳按时，麻糬会产生相应的软体形变与回弹，营造“糯叽叽”的解压手感。 | 基于 2D 柔体物理引擎构建核心骨架，将鼠标输入映射为物理冲量。 |
| 局部组件自定义 | 用户可上传本地图片，自由替换麻糬的五官（如表情）或添加头饰配件。 | 保留标准软体身体，通过 2D 骨骼插槽（Slot）动态加载并挂载外部图片资源。 |

### 2. 日常陪伴与效率组件

- 番茄钟与 ToDo：极简的任务面板。进入专注模式时，DeskMochi 会进入“休眠”或“戴眼镜读书”等安静状态，避免视觉干扰。
- 闹钟与日程：自定义时间提醒，时间到达时宠物会通过跳跃或伸懒腰等动画吸引用户注意。
- 桌面播放器：音乐控制器，播放音乐时宠物会随着节奏产生轻微的呼吸或晃动律动。

### 3. 开发者专属数据驱动

将后台无形的数据转化为前台可见的宠物正向反馈，为工作心流提供情绪激励。

- 打字统计：监听键盘输入频率。敲击越快，麻糬的状态指示条上升，或产生掉落小星星的特效。
- Git 提交响应：监听本地仓库，当完成 `git push` 时，触发专有的庆祝动画（例如举起小旗子或撒花）。
- AI Token 消耗监测：读取本地 AI Agent 的日志文件。随着 Token 消耗的增加，麻糬会展现出“智慧充能”、“超频发热”等极具趣味性的视觉变化。

## 三、技术架构设计规范

为保证桌宠常驻后台时的极低资源占用，并为未来快速迭代打下基础，项目遵循以下技术选型与编程范式：

- 引擎框架：选用 Godot 引擎，利用其对透明无边框窗口及输入穿透的优秀原生支持。
- 架构模式：全面采用面向数据编程（DOP）或 ECS 模式，驱动状态机。摒弃庞大的面向对象继承树。
- 代码约束原则：
  1. 坚决摒弃单例模式，消除全局状态带来的隐式耦合。
  2. 实行严格的分层架构，业务逻辑与渲染表现彻底分离。
  3. 最大限度降低抽象层级。确保代码控制流是单向且有序的，追求静态代码的极致阅读体验。

注：这种清晰、无副作用的架构设计，将极大优化后续使用 AI 助手进行大规模代码生成与迭代（Vibe Coding）的效率。

## 四、开发路线图 (Roadmap)

### Phase 1: 基础架构与软体核心 (M1)

- 搭建引擎的基础透明窗口工程，完善鼠标穿透与基础窗口控制逻辑。
- 实现“麻糬”核心软体物理表现（柔体模拟）。
- 建立基于单向数据流的宠物状态机（包含待机、拖拽、下落、被戳等基础行为）。

### Phase 2: UI 组件与自定义化 (M2)

- 开发轻量级 UI 面板（番茄钟倒计时、基础 ToDo 列表）。
- 实装自定义插槽系统：支持读取外部图片并挂载至面部或头顶。
- 完成本地数据的持久化存储（用户偏好、当前 ToDo 进度）。

### Phase 3: 数据监听集成 (M3)

- 编写本地后台服务，监听键盘输入频率。
- 集成本地 Git 状态读取模块。
- 开发日志解析模块，抓取本地 AI Agent 生成和消耗的 Token 数据。
- 将监听到的外部数据映射回宠物的动画触发系统中。

### Phase 4: 性能打磨与分发准备 (M4+)

- 进行严格的性能 Profile，确保静置和失焦状态下 CPU/GPU 的最小化消耗。
- 拓展内置的默认形象与表现动画。
- 接入分发平台 SDK（如配置成就系统），打包准备发行。
